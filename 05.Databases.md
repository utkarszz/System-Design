## CHALLENGES ##
1. Scalibility : Like ex uber has one db so signup/login, new ride ongoing ride everything is happening in 1 db making things slow so to prevent that having separate db for separate systems is better
2. Consistency : Like ex if one has dbs in 3 different region so if i can run a query on a db of one region then i should also be able to run it on region 2
3. Availability :Your service is avilable most of the time so that people can access it
