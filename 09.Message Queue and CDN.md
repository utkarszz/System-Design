## USING A MESSAGE QUEUE ##
There are two components producers and consumers and message queue lies between them
The role of the producers is they'll keep on queueing the messages in your queue and based on availibility the consumers will start taking the messages one by one

## ADVANTAGES OF A MESSAGE QUEUE ##
1. Decoupling: It allows you to separate components and make them free sooner.
2. The system becomes more scalable as more clients can send in requests asynchronously
3. You can process the message again, if the 1st attempt fails.. This makes the system fault tolerant

## CHALLENGES OF A MESSAGE QUEUE ##
1. Make sure that the ordering of the messages remains same in the order they were received
2. Handling duplicate messages to avoid redundancy and processing cycles
3. Adding something in the middle introduces latency. The benifits should outweigh the lag


## CDN (CONTENT DELIVERY NETWORK ) ##
It stores or caches all of the content you need for the application

## PROS ##
1. Faster load times
2. Reduces latency
3. Caching static content

## CONS ##
1. Your data should always be fresh
2. All the CDN infrastructure is costly
