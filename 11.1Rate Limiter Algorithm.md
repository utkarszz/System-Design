## ALOGORITHMS ##

 ## 1. Token Bucket ##
Theres a bucket and theres a refiller in the bucket(having 3 spaces assume) there are requests that will come to the bucket (assue 5 requests are there) so only 3 request will go forward rest can either be dropped or send back to the client

this method is :-
1. Very easy to implement
2. Memory Efficient
3. Allow bursts of traffic


Issues :-
1.Tuning the refill rate and bucket size is tricky

## 2. Fixed Window Counter ##
So if i set that i need only 3 req per second so any req additional to 3 will be dropped

this method is :-
1. Very easy to implement
2. Memory Efficient
3. Allows you to dynamically manage quota

Issues :-
1. Cannot handle burst of traffic


## 3. Sliding Window Log ##
it will log all the requests for the given time and for the next round it will delete the prev requests

this method is :-
1. Very accurate, and requests are handled correctly in a rolling window

Issues :-
1. Not memory efficient
