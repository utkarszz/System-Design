Caching is just a small db which has the datas that the client use frequently. Its very fast

## BENIFITS ##
1. Increases efficiency of server
2. Saves time of client
3. Increased speed of transaction
4. It just reduces how many times we have to look into the main memory

Example: There is a server for news, there are 4 clients where 3 are asking for article1{article1 is the breaking news} while other are asking for article2. Cache memory already has article one so 3 clients will get it quickly from cache while for one client article2 will be brought from db


## Cache hit : If youre able to find resources in your cache ##
## Cache miss : If youre unable to find resources in your cache ##

HIGHER THE VALUE OF CACHE HIT / CACHE MISS RATIO BETTER IS THE SYSTEM


## CACHE EVICTION METHODS ##
1. Random eviction : Just as the name suggest, used when we have a homogeneous view
2. FIFO:same as the name suggest
3. LFU [LEAST FREQUENTLY USED]: same as the name suggest
4. LRU [LEAST RECENTLY USED]: same as the name suggest

## THINGS TO BE CAREFUL ABOUT ##
1. Consistency of resources: like if someone used article1 from cache memory but now article1 is edited from the og db  so db will have the newer version while cache memory will have the older one
2. Coherence : Caches in different regions could get out of sync.
3. Security : Sessions and credentials should be cached with caution
