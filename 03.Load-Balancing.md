## LOAD BALANCING ##

Example: you own a mall and have one cashier so all the load is on him so which inc waiting time for the rest of the customers making them un happy and to fix it, you can either replace it w a more efficient cashier or add more cashier to distribute the load

Balanccing the load at each of the server is called LOAD BALANCING


## LOAD BALANCER ##

It knows each of the information about each of the server and how many clients are connected to each server

## TYPES OF BALANCING ##

1. Least connections: Clients will be added to the server with min no of clients

2. Round Robin Scheduling: Clients will be alloted to each of the servers one by one

3. Weighted Round Robin Scheduling: Load is balanced on the basis of the servers capacity. More clients are given to efficent servers

4. IP Hashing: Connecting clients to the server closest to their region.


## ADVANTAGES ##
1. Traffic is distributed and processed in parallel
2. Your design becomes easy to scale up
3. High Availability

## DISADVANTAGES ##
1. A load balancer can get very complex and not easy to setup
2. The load balancer itself becomes a signal point of failure
3. If load balancer is compromised, system can go down
